## About

VeneriX is a personal, hobby-built game engine written in modern C++17. It's an exploration of how engines work under the hood — windowing, rendering, audio, input, and a simple scene/entity/component object model — rather than a production-grade framework.

**Built on:**
- **OpenGL 3.3** — rendering
- **GLFW** — windowing & input
- **GLAD** — OpenGL loader
- **GLM** — math
- **Dear ImGui** — debug UI
- **miniaudio** — sound

## Features

- Scene → Entity → Component object model
- Automatic per-frame updates (no manual update loops to write)
- `TimeStep` system with adjustable time scale (pause, slow motion, fast-forward)
- Instanced cube rendering for tiled/voxel-style worlds
- Texture loading (`Texture2D`)
- Simple look-at camera
- Music + SFX playback
- Built-in ImGui integration for debug tooling

## Building the engine

Requires CMake 3.15+, a C++17 compiler, and [vcpkg](https://github.com/microsoft/vcpkg) with `glfw3`, `glad`, `glm`, and `imgui` installed.

```bash
cmake -B build -S . -DCMAKE_TOOLCHAIN_FILE="<path-to-vcpkg>/scripts/buildsystems/vcpkg.cmake"
cmake --build build --config Release
```

Building the `INSTALL` target packages the compiled engine into `VeneriX_SDK/` (headers, import library, and runtime DLL) — ready to be used by a consuming project. See the [VeneriX_SDK](https://github.com/thevenerr/VeneriX_SDK) repository for a ready-to-build starter project using this SDK.
