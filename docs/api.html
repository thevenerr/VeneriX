VeneriX

VeneriX is a lightweight C++ game engine built from scratch as a learning project. It wraps OpenGL rendering, GLFW windowing, and Dear ImGui debug tooling behind a small, scene-based API so you can focus on game logic instead of boilerplate.

C++17 OpenGL 3.3 GLFW GLM Dear ImGui miniaudio

01Quick start

A minimal VeneriX application using the compiled SDK (VeneriX_SDK/include and VeneriX_SDK/lib).

1

Include the engine header

A single header pulls in the whole public API.

#include <VeneriX.h>
#include <EntryPoint.h>
2

Create a scene

Subclass Scene and override Update, Render, and RenderUI as needed.

class MyScene : public Scene {
public:
    MyScene() : Scene("My Scene") {}

    void Render() override {
        // draw your world here
    }
};
3

Create your application

Subclass VeneriX::Application and assign your scene in the constructor.

class MyApp : public VeneriX::Application {
public:
    MyApp(const VeneriX::ApplicationSpecification& spec)
        : VeneriX::Application(spec) {
        SetScene(new MyScene());
    }
};
4

Implement the entry point factory

The engine's main() (defined in EntryPoint.h) calls this function to obtain your application instance.

VeneriX::Application* VeneriX::CreateApplication() {
    VeneriX::ApplicationSpecification spec;
    spec.Name   = "My Game";
    spec.Width  = 1280;
    spec.Height = 720;

    return new MyApp(spec);
}
5

Build and run

Link against venerix (and glm, imgui), make sure venerix.dll and its dependencies (e.g. glfw3.dll) sit next to your executable, then run the binary. The window opens and your scene's Update/Render loop starts automatically.

02Architecture

VeneriX follows a small, explicit object hierarchy rather than a full ECS. Each layer owns the one below it.

Application → Scene → Entity → Component

Application owns the GLFW window, the ImGui context, and the single active Scene. Every frame it computes a TimeStep, then calls, in order: Scene::UpdateEntities (which updates every Entity and in turn every Component attached to it), Scene::Update (your own game logic), and Scene::Render.

Because the engine calls UpdateEntities for you, you never need to manually loop over entities or call Scene::Update on the base class — overriding Update in your scene only needs to contain logic that isn't already handled by your components.

TimeStep & time scale

Every frame, Application computes a TimeStep struct containing both the real (Unscaled) and scaled (Delta) frame time. Calling SetTimeScale changes how fast Delta advances relative to real time — useful for pause menus, slow motion, or fast-forward effects — without affecting Unscaled, which always reflects real elapsed time.

A plain struct describing frame timing, computed once per frame by Application::Run() and passed down through Scene, Entity, and Component.

FieldDefaultDescription

04Examples

Common patterns pulled together into short, focused snippets.

Creating a scene with an entity

class JabaScene : public Scene {
private:
    Entity* m_PlayerEntity;
    StarterPlayer* m_Player;

public:
    JabaScene() : Scene("JabaLevel") {
        m_PlayerEntity = CreateEntity("Player");
        m_Player = m_PlayerEntity->AddComponent<StarterPlayer>(
            glm::vec3(0.0f, 10.0f, 0.0f)
        );
    }
};

Writing a custom component

Components only need to override the hooks they use — both Start() and Update() are optional.

class StarterPlayer : public Component {
public:
    glm::vec3 Position;
    float VelocityY = 0.0f;
    float Gravity = 9.8f;
    bool IsGrounded = false;

    StarterPlayer(glm::vec3 startPos) : Position(startPos) {}

    void Update(const VeneriX::TimeStep& ts) override {
        VelocityY -= Gravity * ts.Delta;
        Position.y += VelocityY * ts.Delta;

        if (Position.y <= 1.0f) {
            Position.y = 1.0f;
            VelocityY = 0.0f;
            IsGrounded = true;
        }
    }

    void Jump() {
        if (IsGrounded) {
            VelocityY = 5.0f;
            IsGrounded = false;
        }
    }
};

Handling input in a scene

void Update(const VeneriX::TimeStep& ts) override {
    if (VeneriX::Input::IsKeyPressed(VENERIX_KEY_SPACE) && m_Player->IsGrounded) {
        m_Player->Jump();
        VeneriX::Audio::PlaySFX("jump.mp3");
    }
}

Controlling time scale

The running Application instance is reachable from anywhere via Application::Get(), so time scale can be changed from a scene, a component, or UI code.

// pause the game
VeneriX::Application::Get()->SetTimeScale(0.0f);

// slow motion
VeneriX::Application::Get()->SetTimeScale(0.3f);

// back to normal
VeneriX::Application::Get()->SetTimeScale(1.0f);

Rendering cubes and textures

void Render() override {
    glm::mat4 view = m_Camera.GetViewMatrix();
    glm::mat4 projection = m_Camera.GetProjectionMatrix();

    // a tiled floor drawn in a single instanced draw call
    VeneriX::Renderer::DrawInstancedCubes(
        grassTexture, floorMatrices.data(), floorMatrices.size(),
        view, projection
    );

    // a single solid-colored player cube
    VeneriX::Renderer::DrawCube(
        m_Player->Position, glm::vec3(1.0f),
        glm::vec3(0.0f, 0.5f, 1.0f),
        view, projection
    );
}

Playing sound

// looping background music, started once in the scene constructor
VeneriX::Audio::PlayMusic("ambient.mp3");

// a one-shot effect, e.g. triggered on jump
VeneriX::Audio::PlaySFX("jump.mp3");
